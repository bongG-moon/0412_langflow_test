# 도메인 등록과 질문 작성 가이드

이 문서는 도메인 지식을 어떻게 추가하고, 어떤 질문을 하면 앱이 더 잘 동작하는지, 그리고 초보자가 어디까지는 코드 없이 확장할 수 있는지를 함께 설명합니다.

## 1. 먼저 구분해야 할 것

새 요구사항이 들어오면 가장 먼저 아래 둘 중 무엇인지 구분하면 됩니다.

### A. 도메인 관리 화면으로 해결 가능한 경우

보통 코드를 수정할 필요가 없습니다.

예:

- 새로운 현장 표현 추가
- 공정 별칭 추가
- 계산 규칙 추가
- 이상/정상 판정 규칙 추가

### B. 개발자가 코드까지 수정해야 하는 경우

예:

- 새로운 데이터셋 추가
- mock 데이터 구조 변경
- 기본 공정 그룹 변경
- 기본 내장 규칙 수정

## 2. 도메인 지식이 필요한 이유

사용자는 항상 시스템이 아는 정식 표현만 쓰지 않습니다.

예:

- `DA공정`
- `후공정A`
- `양품률`
- `생산 달성률`
- `HOLD 이상여부`

이런 표현을 제대로 처리하려면 아래를 알아야 합니다.

- 어떤 데이터셋을 불러와야 하는가
- 어떤 필터 값으로 바꿔야 하는가
- 어떤 계산식이나 판정 규칙을 적용해야 하는가

## 3. 도메인 지식은 두 종류다

### 기본 도메인 지식

코드에 포함된 고정 정보입니다.

- 위치: [`manufacturing_agent/domain/knowledge.py`](../manufacturing_agent/domain/knowledge.py)

포함 내용 예:

- 공정 그룹
- MODE, DEN, TECH, PKG 분류
- 기본 데이터셋 키워드
- 기본 분석 규칙

### 사용자 등록 도메인 지식

도메인 관리 화면에서 사용자가 추가한 정보입니다.

- 위치: [`reference_materials/domain_registry/entries`](../reference_materials/domain_registry/entries)

현재는 JSON 파일 기반입니다.

## 4. 어떤 내용을 등록할 수 있나

### 데이터셋 표현 등록

```text
양품률이라는 표현은 수율 데이터셋을 뜻해.
```

### 필터 별칭 등록

```text
후공정A는 D/A1, D/A2를 뜻해.
```

### 계산 규칙 등록

```text
홀드 부하지수는 hold 테이블의 hold_qty 와 production 테이블의 production 을 같이 가져와서
hold_qty / production 으로 계산해줘.
```

### 판정 규칙 등록

```text
장비 가동 이상여부는 equipment 테이블의 가동률이 85 미만이면 이상,
아니면 정상으로 판단해줘.
```

## 5. 좋은 등록 문장을 만드는 팁

가능하면 아래 정보를 함께 넣어 주세요.

- 어떤 데이터셋을 쓰는지
- 어떤 컬럼을 쓰는지
- 계산식이 무엇인지
- 조건식이 무엇인지
- 결과 컬럼 이름이 무엇인지
- 기본 집계 기준이 무엇인지

예:

```text
생산 목표 차이율은 production 테이블의 production 값과 target 테이블의 target 값을 가져와서
(production - target) / target 으로 계산해줘.
기본 집계 기준은 공정으로 해줘.
결과 컬럼 이름은 plan_gap_rate 로 해줘.
```

## 6. 등록 절차

추천 순서는 아래와 같습니다.

1. `도메인 관리`로 이동합니다.
2. 줄글 설명을 입력합니다.
3. `미리보기`를 눌러 구조화 결과를 확인합니다.
4. 경고/오류를 읽습니다.
5. 문제 없으면 `등록`합니다.
6. `채팅 분석` 화면으로 돌아가 실제 질문으로 테스트합니다.

## 7. 미리보기에서 꼭 봐야 할 것

- 제목
- 데이터셋 키워드
- 필터 값 그룹
- 분석 규칙
- 경고/오류

특히 분석 규칙에서는 아래 필드를 보면 좋습니다.

- `required_datasets`
- `required_columns`
- `source_columns`
- `calculation_mode`
- `condition`
- `decision_rule`
- `output_column`

## 8. 경고와 오류는 어떻게 해석하나

### warning

저장은 가능하지만 의미가 기존 것과 겹치거나 애매할 수 있다는 뜻입니다.

예:

- 비슷한 keyword가 이미 존재함
- 계산 규칙인데 결과 컬럼 이름이 부족함
- condition 문장이 너무 빈약함

### error

실제 충돌이 있어 저장하면 안 되는 경우입니다.

예:

- 기존 의미와 다른데 같은 이름이나 별칭을 쓰는 경우
- unsafe text가 들어간 경우

## 9. 좋은 질문의 기본 원칙

아래 네 가지가 들어가면 대부분 잘 동작합니다.

- 언제
- 어디서
- 무엇을
- 어떻게 보고 싶은지

예:

```text
어제 DA공정에서 DDR5제품 생산 달성률 알려줘
```

## 10. 질문 예시

### 기본 조회

- `오늘 DA공정 생산량 알려줘`
- `어제 WB공정 수율 보여줘`
- `오늘 DDR5제품 WIP 알려줘`

### 비교 조회

- `오늘 생산과 목표를 같이 보여줘`
- `오늘 생산과 재공을 같이 보여줘`
- `어제와 오늘 DA공정 생산량을 공정별로 비교해줘`

### 집계/후처리

- `오늘 DA공정에서 MODE별 생산량 알려줘`
- `오늘 DA공정에서 세부 공정별 생산량 비교해줘`
- `MODE별로 묶어줘`
- `상위 5개만 보여줘`

### 파생 지표

- `어제 DA공정 생산 달성률 알려줘`
- `오늘 DA공정 홀드 부하지수 알려줘`
- `오늘 DA공정 HOLD 이상여부 알려줘`

## 11. 등록 후 테스트 예시

### 데이터셋 표현 등록 후

등록:

```text
양품률이라는 표현은 수율 데이터셋을 뜻해.
```

질문:

```text
오늘 DA공정 양품률 알려줘
```

### 필터 별칭 등록 후

등록:

```text
후공정A는 D/A1, D/A2를 뜻해.
```

질문:

```text
오늘 후공정A 생산량 알려줘
```

### 계산 규칙 등록 후

등록:

```text
생산 달성율은 production 테이블의 production 값과 target 테이블의 target 값을 가져와서
production / target 으로 계산해줘.
```

질문:

```text
어제 DA공정에서 DDR5제품 생산 달성률 알려줘
```

## 12. 한 줄 요약

이 프로젝트는 많은 경우 코드 수정 없이도 도메인 관리 화면과 더 명확한 질문 작성만으로 의미 있는 확장이 가능하며, 정말 구조가 바뀔 때만 코드를 수정하면 됩니다.
